# Koopify

A little web app to transfer funds from your wallet to mine and transfer snake oil from my non-existant inventory to yours!

## Demoing the code

For demoing a production deployment, visit <https://koopify.piguy.nl>.
Docker compose can be used to demo the code locally.

Version of docker-compose used for testing:
```sh
$ docker-compose --version
Docker Compose version 5.1.1
```

### Docker-Compose Endpoints
- Frontend: <https://localhost:5173> (uses TLS)
- Backend: <http://localhost:8080>

### Credentials
New users can be created in the frontend. An auto-generated admin user is also created:
- Email: `admin@koopify.piguy.nl`
- Password: `admin123`

### Example dataset
The `compose.yaml` file takes care of migrations. An example dataset of Gen1
Pokemon cards can be used to fill the database with "products". The docker
compose manifest does also add this dataset to the database, however, it can
also be manually added if so desired:

```sh
cd example-dataset
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# these scripts actually run the sql queries
python3 insert_products.py --csv gen1.csv --db <database url>
python3 insert_categories.py --csv gen1.csv --db <database url>
```

## Environment variables

### Backend (API)

Required secrets/config:

- `JWT_SECRET`: Secret used to sign JWTs. Must be a non-default, non-empty value.
- `PGDB`: Postgres connection string. Default: `postgres://postgres:postgres@localhost:5432/?sslmode=disable`
- `CHECKOUT_RETURN_URL`: Adyen redirects back to this page after payment
- `ADYEN_API_KEY`: The API key for Adyen
- `ADYEN_MERCHANT_ACCOUNT`: The merchant account for Adyen
- `ADYEN_THEME_ID`: The theme ID for Adyen

Optional secrets/config:

- `HOST_ADDR`: Bind address for the API server. Default: `:8080`
- `TLS_ENABLED`: Enable TLS when set to a truthy value (any non-empty value other than `0`).
- `TLS_CERT`: Path to TLS certificate file (required when `TLS_ENABLED` is on).
- `TLS_KEY`: Path to TLS private key file (required when `TLS_ENABLED` is on).

### Frontend (Web)

Required config:

- `VITE_API_BASE_URL`: Base URL for the API (e.g. `https://localhost:8080`).

This variable neds to be present at build-time, and not runtime.


## AI usage disclaimer

Generative AI has been used to design the front-end and produce the style
sheets for the frontend. It has also been used to alter parts of the code, be
it replicating schema from the backend into TypeScript types in the front-end,
or making adjustments to the UI/layout of the front-end page. It has also been
used to write some of the business logic, but most of it is hand-written or
generated by the ORM used in this project (the ORM does not use Generative AI).

AI was also used in the form of autocomplete. I use supermaven in my text
editor, and use the key combination of `<leader>ai` to toggle the AI-assisted
completions. This was mostly used when writing the frontend code, to get more
context-aware suggestions, as the typescript language server is subpar.

For all my other usages of AI LLMs, I relied on Opencode, with the
MiniMax-M2.5, MiniMax-M2.7 and Kimi-K2.5 models.
